# CH1 - (In)segurança de Aplicações Web
- quando usuários podem fornecer inputs livremente surge um problema core de segurança -> entradas específicas podem deixar o programa vulnerável

## categorias de vulnerabilidades

- broken authentication: defeitos no mecanismo de login (brute-force, bypass login)

- broken access control: acesso de dados indevidos

- sql injection: executar comandos no db

- cross-site scripting

- information leakage: divulgação de dados não necessários que ajudam o atacante, alguns error handling

- cross-site request forgery: enganar o browser de forma que ele realize ações (requests) não desejadas pelo usuário
